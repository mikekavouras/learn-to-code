Accessing elements in an array

In order to remember which of our friends are allergic to nuts, we need a way to retrieve the elements from our array. The elements of an array are accessed with a special property called an index. An index is nothing more than the numeric position of an element. The only catch is that this index doesn't start at 1, it starts at 0.

Let's try and access the first person who is allergic to nuts and assign it to a new variable, firstAllergicPerson. 


var firstAllergicPerson = peopleAllergicToNuts[0]; // sam

Notice the syntax. We're using the name of the variable referencing our array, peopleAllergicToNuts followed by [0]. The brackets tell our array that we're attempting to access an element. The 0 is the index of the element we're attempting to access (remember, our indices begin at 0, not 1). 


peopleAllergicToNuts[0]; // sam
peopleAllergicToNuts[1]; // alex
peopleAllergicToNuts[2]; // dylan
peopleAllergicToNuts[3]; // hunter
peopleAllergicToNuts[4]; // ?????? (UH OH)

Adding elements to an array

Often you'll find yourself wanting to add new elements to an array. Say we discover than our other friend, Casey, is also allergic to nuts. There are a few ways add new elements to an array. One way is to use an explicit index. 


peopleAllergicToNuts; // [sam, alex, dylan, hunter]

var casey = new Person();
peopleAllergicToNuts[4] = casey;
peopleAllergicToNuts; // [sam, alex, dylan, hunter, casey]

The safer way to add an element is by asking the array to add it for you. To do this you ask the array to "push" a new element "on top" of the current list. 


peopleAllergicToNuts; // [sam, alex, dylan, hunter, casey]

var taylor = new Person();
peopleAllergicToNuts.push(taylor);
peopleAllergicToNuts; // [sam, alex, dylan, hunter, casey, taylor]

Generally, arrays are smarter than they appear. They have the ability to manipulate themselves, like adding or removing elements, as well as the ability to communicate information that we may find useful. For instance, an array has the ability to tell us how many elements it has at any given moment. We can rely on these features to build safer code.

Changing elements in an array

There will be times when you need to change an existing element in an array. Say, for instance, our buddy Dylan has suddenly been cured, and at the exact same moment we found out that our friend Blake is horribly allergic and will die at the simple touch of a nut.

Let's use what we know to replace Dylan with Blake. 


peopleAllergicToNuts; // [sam, alex, dylan, hunter, casey, taylor];

var blake = new Person();
peopleAllergicToNuts[2] = blake;
peopleAllergicToNuts; // [sam, alex, blake, hunter, casey, taylor];

Notice how the third element (index 2), dylan, was replaced by our new friend, blake.
